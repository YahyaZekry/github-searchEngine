## 4. Responsive Design Breakpoints

### 4.1 Desktop (1200px+)

- 4-column grid layout
- Full filter panel visible
- Rich user cards with all features
- Horizontal pagination

### 4.2 Tablet (768px - 1199px)

- 2-3 column grid layout
- Collapsible filter panel
- Slightly simplified user cards
- Horizontal pagination with fewer page numbers

### 4.3 Mobile (320px - 767px)

- Single column layout
- Slide-out filter panel
- Stacked user cards
- Vertical pagination buttons
- Touch-optimized interactions

## 6. Animation Specifications

### 6.1 Page Transitions

- **Fade In**: 300ms ease-out
- **Slide Up**: 250ms ease-out
- **Scale In**: 200ms ease-out

### 6.2 Micro-interactions

- **Button Press**: 100ms ease-in-out
- **Card Hover**: 200ms ease-out
- **Dropdown**: 150ms ease-out
- **Modal**: 300ms ease-out

### 6.3 Loading Animations

- **Skeleton Pulse**: 1.5s infinite
- **Spinner**: 1s linear infinite
- **Progress Bar**: 2s ease-in-out

This visual specification provides a comprehensive guide for implementing a modern, user-friendly interface that enhances the GitHub Users Search Engine experience.
